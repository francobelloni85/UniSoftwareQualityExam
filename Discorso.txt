1) the purpose of this project is to analyze a dataset and find out if there are any correlations between bugs in the software and the metrics used to measure the software itself 

2) the first step is to understand the data. The dataset is composed of 24 columns that indicate the 24 characteristics of a single module and 745 rows, 
For the project we focus on the bug entry 

3) To choose the best features we use the correlation matrix. The values of the matrix indicate when 2 features are related to each other. The values can range from -1 to 1. 1 indicates maximum correlation while 0 indicates no correlation.
